# Working rules

- Keep the core slot-based. Do not hardcode header/footer semantics into `ModularDialog`.
- Add concrete UI structures as shell plugins or presets.
- Keep all source as Vanilla JavaScript ES modules.
- Keep comments and documentation in English.
- Use tabs for indentation and opening braces on the same line.
- Preserve dependency-free browser usage.
- Use commands for imperative API extensions.
- Use cancelable lifecycle events for vetoable behavior.
- Keep display-specific logic outside this package.
